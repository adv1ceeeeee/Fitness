# Sportify — Описание функционала

## Архитектура

### Анимированные переходы
- Экраны авторизации / онбординга: `FadeTransition` (220 мс)
- Вторичные экраны (сессия, программы, аналитика и т.д.): slide-up + fade (`Offset(0, 0.07)` → `Offset.zero`, 280 мс / 220 мс обратно)
- Корневые табы навигации: `NoTransitionPage` (мгновенно)

### Офлайн-режим
- `connectivity_plus` → `connectivityProvider` (StreamProvider)
- Оранжевый баннер в `MainShell` при потере соединения (AnimatedContainer)
- `CacheService`: кешируем список программ (`workouts.json`) и тренировку на сегодня (`today_workout.json`)
- Fallback при сетевой ошибке: данные из кеша с отображением метки «данные из кеша» и деактивацией кнопки «Начать»



| Слой | Технология |
|---|---|
| UI | Flutter (Material + Cupertino) |
| Маршрутизация | GoRouter (ShellRoute для persistent nav) |
| Состояние | Riverpod (StateNotifierProvider) |
| Бэкенд | Supabase (Auth, PostgreSQL, Storage) |
| Хранилище на устройстве | FlutterSecureStorage, SharedPreferences |
| Криптография | HMAC-SHA256 (пакет `crypto`) |
| Автодополнение городов | Dadata API |

Директория `lib/`:
```
lib/
├── config/         # Темы, роутер, конфиг (API-ключи)
├── data/           # Статичные данные (стандартные программы)
├── models/         # Dart-классы с fromJson/toJson
├── providers/      # Riverpod providers
├── screens/        # Экраны приложения
│   ├── auth/
│   ├── analytics/
│   ├── calendar/
│   ├── home/
│   ├── onboarding/
│   ├── profile/
│   ├── workout_session/
│   └── workouts/
└── services/       # Слой доступа к данным (Supabase + устройство)
```

## Аутентификация и онбординг

### WelcomeScreen
Первый экран при запуске.

- При инициализации проверяет `PinService.hasPin()`:
  - Если PIN найден (локально или в Supabase metadata) → сразу переходит на `/pin-login`
  - Иначе — показывает экран с двумя кнопками
- Кнопки: **Войти с паролем** → `/login`, **Создать аккаунт** → `/register`

### LoginScreen
- Поля: логин (email или никнейм) + пароль
- При вводе никнейма → RPC `get_email_by_nickname` конвертирует его во внутренний email
- Ошибки: неподтверждённый email, неверные данные, никнейм не найден
- Ссылка **Войти по PIN-коду** (если PIN настроен)
- Ссылка **Создать** → `/register`

### RegisterScreen
- Поля: никнейм (3–32 символа: a–z, 0–9, `_`, `.`), пароль (мин. 6 символов), подтверждение пароля
- Чекбокс **Запомнить меня на этом устройстве**
- При успехе:
  - Чекбокс включён → `/pin-setup`
  - Чекбокс выключен → `/onboarding-check`

### PinSetupScreen
- Двухшаговый ввод PIN-кода (4 цифры + подтверждение)
- При совпадении → `PinService.setupPin()` → сохраняет в SecureStorage и Supabase user_metadata → `/onboarding-check`
- Кнопка **Пропустить** → `/onboarding-check`

### PinLoginScreen
- Показывает аватар пользователя и «Привет, [никнейм]!»
- Ввод PIN → `PinService.verifyPin()`
- После 5 неудачных попыток — блокировка (кнопки неактивны)
- Ссылка **Войти с паролем** → `/login`
- При успехе и живой Supabase-сессии → `/onboarding-check`

### OnboardingCheckScreen
- Проверяет, заполнен ли профиль (наличие `profiles.goal`)
- Да → `/home`, Нет → `/onboarding`

### OnboardingScreen
Трёхстраничный PageView с прогресс-индикатором:

| Страница | Содержимое |
|---|---|
| 1 | Пол (М/Ж), Возраст, Вес (кг) |
| 2 | Цель: Силовая / Похудение / Набор массы / Выносливость |
| 3 | Уровень: Новичок / Средний / Продвинутый |

- Кнопки **Далее** / **Начать**
- **Пропустить** → `/home` без сохранения данных
- Сохранение → `ProfileService.updateProfile()`

## Главный экран

### HomeScreen
- Приветствие: «Привет, [имя]!»
- Карточка **Тренировка на сегодня** → `/today`
- Карточка **Чек-ин**: вес тела (кг) + метрики самочувствия (сон, стресс, энергия) → сохраняется в `body_metrics` и `wellness_logs`
- Быстрые ссылки: Мои программы → `/workouts`, Аналитика → `/analytics`, Календарь → `/calendar`
- Данные: загружает профиль через `ProfileService.getProfile()`

**Карточка «Прогресс тела»** (на главном экране)
- Выпадающий список метрик: Вес · % жира · Талия · Грудь · Бёдра · Бицепс · Плечи
- Плитка «Фактический»: последнее значение + дата и время замера (из `body_metrics.updated_at`)
- Плитка «Цель»: устанавливается через диалог, хранится в SharedPreferences с ключом `goal_{metric}_{userId}`
- Строка прогресса: «До цели: ±N кг» / «Цель достигнута!» / «Нажмите на «Цель» для установки»
- Баннер достижения: «🏆 Цель по {метрике} за X дн./нед./мес.» (анимированно появляется при достижении цели)

**Онбординг**
- Показывается ровно один раз: флаг хранится в `SharedPreferences` (ключ `onboarding_shown_{userId}`) и дублируется в `user_events` (событие `onboarding_shown`)
- При первом запуске на новом устройстве/браузере проверяет `user_events` в Supabase как fallback, чтобы онбординг не повторялся

### TodayScreen
Показывает программу на сегодня.

- Нет тренировки → «Сегодня нет запланированной тренировки» + ссылка на программы
- Есть тренировка:
  - Название, список упражнений (порядок, название, подходы × повторения)
  - Кнопка **Начать тренировку** → `TrainingService.getOrCreateTodaySession()` → `/session/{id}`

## Программы тренировок

### WorkoutsScreen
Два таба:

**«Мои программы»**
- Два подраздела: **Действующие программы** (есть дни недели) и **Завершённые / неактивные** (дни не назначены)
- Действующие: карточки перетаскиваемые (drag-to-reorder) — `WorkoutService.reorderWorkouts()`
- Неактивные: показывают дату и продолжительность последней сессии
- Свайп влево → скрыть / удалить
- Кнопка **Создать программу** → `/workouts/create`
- Карточка программы: название + «X тренировок в неделю»
- Нажатие → `/workouts/{id}/exercises`
- Пустое состояние «Нет программ тренировок» — выровнено по центру экрана

**«Стандартные»**
- 4 готовых шаблона (подробнее в разделе [Стандартные программы](#стандартные-программы))
- Нажатие → превью с упражнениями
- Кнопка **Использовать эту программу** → копирует программу со всеми упражнениями в «Мои программы»

### CreateWorkoutScreen
Форма создания программы:

- Поле «Название» (например, «Ноги», «Грудь+бицепс»)
- Выбор дней недели (Пн–Вс, множественный выбор)
- Слайдер **Длительность цикла** (4–16 недель, шаг 1)
  - Метка со значением отображается над ползунком
  - Двойное нажатие → ручной ввод (можно выйти за диапазон, например 20 недель)
- Кнопка **Далее** → `WorkoutService.createWorkout()` → `/workouts/{id}/exercises`
- Валидация: обязательное название и хотя бы один день

### AddExercisesScreen
Экран настройки упражнений в программе.

**Шапка**
- Название программы
- Иконка редактирования (карандаш) → диалог редактирования (название, дни, цикл)

**Секция «В программе»**
- Список добавленных упражнений с drag-and-drop для изменения порядка
  - Рукоятка перетаскивания (⠿) слева
  - `WorkoutService.reorderExercises()` при отпускании
- Нажатие на упражнение → нижний лист настроек:
  - Количество подходов (±1, минимум 1)
  - Количество повторений (текстовое поле)
  - Целевой вес (кг, необязательно)
  - Слайдер **Отдых** (0–120 сек, двойное нажатие → ручной ввод без ограничений)
  - Сохранение → `WorkoutService.updateWorkoutExercise()`
- Иконка корзины → удаление → `WorkoutService.removeExerciseFromWorkout()`

**Суперсеты**
- Кнопка «Связать» между соседними упражнениями в списке → объединяет их в суперсет
- Упражнения одного суперсета получают одинаковый `superset_group` (целочисленный ID)
- Цветные бейджи «A1 / A2 / B1 / B2…» + цветная левая граница карточки
- Повторное нажатие — разрывает связь

**Секция «Добавить упражнение»**
- Поле поиска с debounce 300 мс
- При активном поиске — плоский список с фильтрацией по названию
- В режиме обзора — упражнения сгруппированы по категориям мышц:
  - Порядок категорий: Грудь · Спина · Плечи · Руки · Ноги · Кардио
  - Каждая категория предваряется заголовком-разделителем
- Нажатие «+» → нижний лист настроек (аналогичен редактированию, дефолты: 3 подхода, 8–12 повт., 90 сек)
  - Сохранение → `WorkoutService.addExerciseToWorkout()`

## Сессия тренировки

### WorkoutSessionScreen
Основной рабочий экран во время тренировки.

**Прогресс-бар**
- `LinearProgressIndicator` в `AppBar.bottom` (высота 4 dp)
- Значение = (завершённые подходы всех пройденных упражнений + завершённые подходы текущего упражнения) / общее кол-во подходов тренировки

**Навигация по упражнениям**
- Счётчик «X / Y» (текущее / всего)
- Название упражнения крупно
- «X из Y подходов выполнено»
- Подсказка **«Прошлый: N кг × M (dd.MM)»** — данные из последней завершённой сессии через `AnalyticsService.getLastSetsForExercises()`

**Таблица подходов**
Для каждого подхода:
- Номер подхода (бейдж)
- Поле ввода веса (текстовое, предзаполняется весом из последней сессии)
- Степпер повторений (±1)
- Степпер RPE (0–10, «—» для пустого)
- Кнопка-чекбокс → отмечает подход выполненным:
  - `HapticFeedback.lightImpact()` при нажатии
  - Мгновенное обновление UI (оптимистичный апдейт)
  - Сохранение `TrainingService.saveSet()` → при сбое показывает SnackBar «Не удалось сохранить подход» с кнопкой **Повторить**
  - Запускает таймер отдыха

**Суперсеты в сессии**
- Если следующее упражнение в том же суперсете — переход без экрана отдыха
- Бейдж «Суперсет A · 1/2» отображается над названием упражнения

**Фазы разминки и заминки**
- Если у программы задан `warmup_minutes > 0` — перед упражнениями показывается экран обратного отсчёта разминки
- После последнего упражнения — аналогичный экран заминки (`cooldown_minutes`)
- Настраиваются в диалоге редактирования программы

**Кнопки действий**
- **Добавить подход** — добавляет строку в таблицу
- **Следующее упражнение** — появляется когда все подходы выполнены и это не последнее упражнение

**Личный рекорд**
- При завершении подхода сравнивает вес с `TrainingService.getPersonalBest()`
- Если новый рекорд — SnackBar «Личный рекорд! 🏆» + событие в `user_events`

**Экран отдыха**
- Счётчик времени (формат ММ:СС), считает вверх от 0
- Продолжительность целевого отдыха берётся из настроек программы (`restSeconds`)
- Кнопка **Готов** (= «Пропустить») — немедленный переход
- `HapticFeedback.mediumImpact()` при завершении отдыха
- **Пауза при сворачивании**: при уходе в фон (`AppLifecycleState.paused`) таймер останавливается; при возврате (`resumed`) прибавляет прошедшее время

**Выход**
- Кнопка «×» / свайп назад (`PopScope`) → диалог «Прервать тренировку?»
  - **Продолжить** — возврат в сессию
  - **Выйти** — останавливает `activeSessionProvider`, переход на `/home`

### SessionSummaryScreen
Экран итогов после остановки тренировки.

**Шапка**
- Иконка трофея, «Тренировка завершена!»
- Плитки: общее время · количество упражнений · количество подходов

**Таблица результатов**
- По каждому упражнению: карточка с упражнением и строками по подходам
- Строка подхода: [Номер] [Вес × Повт.] [RPE, если есть] [Иконка редактирования]
- Нажатие на строку → нижний лист редактирования:
  - Поля «Вес (кг)» и «Повторения»
  - Выбор RPE: чипы «—», 6, 7, 8, 9, 10
  - Кнопка **Применить** → немедленный `TrainingService.updateSet()` (автосохранение правки)

**Сохранение**
- Кнопка **Сохранить тренировку**:
  1. Валидация: если у подхода `reps == null` или `reps == 0` — диалог «Не указаны повторения» со списком проблемных подходов и выбором «Отмена» / «Сохранить всё равно»
  2. Сохраняет все изменённые подходы через `TrainingService.updateSet()`
  3. Помечает сессию завершённой: `TrainingService.completeSession()` с `durationSeconds`
  4. Сбрасывает `activeSessionProvider`
  5. Переход на `/home`
- Кнопка «×» / назад → диалог «Выйти без сохранения?»

## Календарь

### CalendarScreen
- Навигация по месяцам (← →)
- Виджет `TableCalendar` (локаль: русский, неделя с понедельника)

**Цветовое кодирование дат**
| Индикатор | Значение |
|---|---|
| Большая синяя заливка | Выбранная дата |
| Серая заливка | Сегодня |
| Синяя точка под датой | Тренировка выполнена |
| Полупрозрачная точка | Тренировка запланирована / разовая сессия |

**Легенда** — «Выполнено» / «Запланировано»

**Взаимодействие**
- Одиночное нажатие → выбирает дату, показывает список событий ниже
- **Длинное нажатие / двойное нажатие** → нижний лист с действиями:
  - Список программ для назначения разовой тренировки на этот день → `TrainingService.scheduleSession()`
  - Предупреждение при конфликте расписания (программа уже запланирована циклом)
  - Кнопка **Создать программу** → `/workouts/create`

**Данные**
- Завершённые сессии: `TrainingService.getSessionsByDateRange()`
- Запланированные разовые сессии: хранятся в `training_sessions` (completed=false)
- Будущие цикличные даты: вычисляются из `workout.days` + `cycleWeeks`
- Конфликты: разовая сессия и цикличная на одну дату — оба показываются

## Аналитика

### AnalyticsScreen
- Приветствие «Привет, [имя]!»
- Отображение цели пользователя («Твоя цель: …»)

**Карточка стрика**
- 🔥 «Стрик: X дней» (последовательные дни тренировок)
- «Тренировок всего: Y»

**Статистика за неделю** (текущая неделя, Пн–Вс)
- Количество тренировок
- Объём (кг) = сумма (вес × повторения) по всем подходам

**Прогресс по упражнению**
- Список упражнений, по которым когда-либо фиксировался вес: `AnalyticsService.getTrackedExercises()`
- При выборе упражнения — график максимального веса по датам: `AnalyticsService.getExerciseMaxWeight()`

**Динамика параметров тела**
- Выпадающий список для выбора метрики: Вес · Шея · Плечи · Грудь · Талия · Бёдра · Бёдро лев./пр. · Голень лев./пр. · Предплечье лев./пр.
- В списке показываются только метрики, по которым есть хотя бы одна запись
- Данные из `body_metrics`, сортировка по дате

**Баланс мышечных групп**
- Круговая диаграмма (`PieChart`): доля завершённых подходов по группам мышц
- Легенда справа от диаграммы; вся секция центрирована по горизонтали

**История тренировок**
- Кнопка «История тренировок» → `/history`
- Все завершённые сессии в обратном хронологическом порядке, сгруппированные по месяцам
- Карточка: число, день недели, название программы, продолжительность, заметка
- Нажатие на карточку → анимированное раскрытие (AnimatedSize) с деталями подходов, сгруппированных по упражнениям
- Подходы отображаются чипами: «1. 80 кг × 10», «2. 80 кг × 10» и т.д.

**Достижения**
- Сетка 3×N бейджей: разблокированные — цветные с синей рамкой, заблокированные — серые
- 6 тренировочных милестоунов (1 / 5 / 10 / 25 / 50 / 100 тренировок)
- 5 серийных милестоунов (3 / 7 / 14 / 30 / 100 дней подряд)
- Tooltip с описанием каждого достижения

**Поделиться прогрессом**
- Карточка с градиентным фоном: имя, стрик, тренировочная статистика
- Кнопка «Поделиться картинкой» — захватывает карточку через `RepaintBoundary.toImage()` (PNG 3×), шарит через `Share.shareXFiles`

**Источники данных**
- `AnalyticsService.getTotalWorkouts()` — всего завершённых сессий
- `AnalyticsService.getBestStreak()` — наибольшее количество последовательных дней

- `AnalyticsService.getVolumeThisWeek()` — объём за текущую неделю
- `AnalyticsService.getTrackedExercises()` — список упражнений с весовыми данными
- `AnalyticsService.getLastSetsForExercises()` — последний подход для каждого упражнения

## Ключевые сценарии использования

### Регистрация и первый вход
```
Добро пожаловать
  → Создать аккаунт (никнейм + пароль)
  → [Настройка PIN, если "Запомнить меня"]
  → Онбординг: пол/возраст/вес → цель → уровень
  → Главный экран
```

### Повторный вход
```
Добро пожаловать
  → Если PIN настроен → PIN-экран → Главный экран
  → Иначе → Вход по паролю → Главный экран
```

### Создание программы тренировок
```
Программы → Создать программу
  → Название + дни недели + длительность цикла
  → Экран упражнений: добавить из каталога (с группировкой по мышцам),
    настроить подходы/повт./отдых
  → Изменить порядок перетаскиванием
```

### Проведение тренировки
```
Главная → Тренировка на сегодня → Начать
  или FAB ▶ (если программа назначена на сегодня)
  → Для каждого упражнения:
      Ввести вес (предзаполнен из прошлой сессии) →
      Отметить подходы (повт. + RPE) →
      Отдых (счётчик вверх) → Следующее
  → Стоп → Итоги: проверить/отредактировать → Сохранить
```

### Просмотр истории
```
Аналитика → «История тренировок» → /history
  Список завершённых сессий, группировка по месяцам
  Карточка: дата · день недели · программа · длительность · заметка
Календарь → Нажать дату → Список тренировок / запланировать разовую
Аналитика → Стрик, всего тренировок, объём за неделю, прогресс по упражнению, достижения
```

- Поле «Вес (кг)» + слайдер повторений (1–20)
- Поле **Вес тела (кг)**: предзаполняется автоматически из `BodyMetricsService.getLatest()` при открытии
- Результат: крупно «Расчётный 1ПМ (среднее)» + «Среднее по N формулам»
- Коэффициент от веса тела: «×3.40 веса тела» (× первым, потом коэффициент)
- Таблица по формулам: Эпли · Бжицки · О'Коннер / Мэйхью / Ломбарди · Ватен / Ландер
- Таблица «Проценты от 1ПМ»: 60–100% с рекомендуемым количеством повторений

**Вкладка «Блины»**
- Единицы: кг / lb; выбор грифа (20 / 15 / 10 / 7.5 кг или 45 / 35 / 25 / 15 lb)
- Поле «Целевой вес»; показывает «Фактический вес» и «Блинов на сторону»
- **4 варианта «На каждую сторону»**: основной блин 25 / 20 / 15 / 10 кг + добор меньшими; строка скрывается, если основной блин не помещается
  - Каждый вариант: цветной бейдж основного блина + чипы с группировкой `N×W кг`
  - Цвета по стандарту IPF: 25 кг = красный, 20 = синий, 15 = жёлтый, 10 = зелёный и т.д.
- **Схема разминки** («по стандарту NSCA/IPF»):
  1. **Суставная разминка**: резинки / гантели / гриф без блинов (зелёный бейдж)
  2. **Общая разминка**: гриф × 10 повт.
  3–N. **Прогрессивные подходы**: рассчитываются от ratio `target/bar`:
     - ratio < 1.5 → только суставная + гриф
     - ratio 1.5–2.5 → +1 подводящий (60%)
     - ratio 2.5–4.0 → специфическая (45%) + подводящий (75%)
     - ratio 4.0–6.0 → 2 специфических + подводящий
     - ratio ≥ 6.0 → 3 специфических + подводящий
  - Веса округляются до 2.5 кг (5 lb), дубликаты пропускаются
  - В конце — рабочий подход с иконкой гантели

**Чистые функции** (top-level, покрыты unit-тестами):
- `calculatePlates(target, barWeight, availablePlates)` — жадный алгоритм
- `calculatePlatesWithPrimary(target, barWeight, primary, availablePlates)` — алгоритм с доминирующим блином
- `groupPlates(plates)` — группировка в `{plate: count}`
- `buildWarmupSets(target, barWeight, useKg)` — схема разминки
