# Sportify — Описание функционала

## Архитектура

| Слой | Технология |
|---|---|
| UI | Flutter (Material + Cupertino) |
| Маршрутизация | GoRouter (ShellRoute для persistent nav) |
| Состояние | Riverpod (StateNotifierProvider) |
| Бэкенд | Supabase (Auth, PostgreSQL, Storage) |
| Хранилище на устройстве | FlutterSecureStorage, SharedPreferences |
| Криптография | HMAC-SHA256 (пакет `crypto`) |
| Автодополнение городов | Dadata API |

Директория `lib/`:
```
lib/
├── config/         # Темы, роутер, конфиг (API-ключи)
├── data/           # Статичные данные (стандартные программы)
├── models/         # Dart-классы с fromJson/toJson
├── providers/      # Riverpod providers
├── screens/        # Экраны приложения
│   ├── auth/
│   ├── analytics/
│   ├── calendar/
│   ├── home/
│   ├── onboarding/
│   ├── profile/
│   ├── workout_session/
│   └── workouts/
└── services/       # Слой доступа к данным (Supabase + устройство)
```

## Аутентификация и онбординг

### WelcomeScreen
Первый экран при запуске.

- При инициализации проверяет `PinService.hasPin()`:
  - Если PIN найден (локально или в Supabase metadata) → сразу переходит на `/pin-login`
  - Иначе — показывает экран с двумя кнопками
- Кнопки: **Войти с паролем** → `/login`, **Создать аккаунт** → `/register`

### LoginScreen
- Поля: логин (email или никнейм) + пароль
- При вводе никнейма → RPC `get_email_by_nickname` конвертирует его во внутренний email
- Ошибки: неподтверждённый email, неверные данные, никнейм не найден
- Ссылка **Войти по PIN-коду** (если PIN настроен)
- Ссылка **Создать** → `/register`

### RegisterScreen
- Поля: никнейм (3–32 символа: a–z, 0–9, `_`, `.`), пароль (мин. 6 символов), подтверждение пароля
- Чекбокс **Запомнить меня на этом устройстве**
- При успехе:
  - Чекбокс включён → `/pin-setup`
  - Чекбокс выключен → `/onboarding-check`

### PinSetupScreen
- Двухшаговый ввод PIN-кода (4 цифры + подтверждение)
- При совпадении → `PinService.setupPin()` → сохраняет в SecureStorage и Supabase user_metadata → `/onboarding-check`
- Кнопка **Пропустить** → `/onboarding-check`

### PinLoginScreen
- Показывает аватар пользователя и «Привет, [никнейм]!»
- Ввод PIN → `PinService.verifyPin()`
- После 5 неудачных попыток — блокировка (кнопки неактивны)
- Ссылка **Войти с паролем** → `/login`
- При успехе и живой Supabase-сессии → `/onboarding-check`

### OnboardingCheckScreen
- Проверяет, заполнен ли профиль (наличие `profiles.goal`)
- Да → `/home`, Нет → `/onboarding`

### OnboardingScreen
Трёхстраничный PageView с прогресс-индикатором:

| Страница | Содержимое |
|---|---|
| 1 | Пол (М/Ж), Возраст, Вес (кг) |
| 2 | Цель: Силовая / Похудение / Набор массы / Выносливость |
| 3 | Уровень: Новичок / Средний / Продвинутый |

- Кнопки **Далее** / **Начать**
- **Пропустить** → `/home` без сохранения данных
- Сохранение → `ProfileService.updateProfile()`

## Главный экран

### HomeScreen
- Приветствие: «Привет, [имя]!»
- Карточка **Тренировка на сегодня** → `/today`
- Быстрые ссылки: Мои программы → `/workouts`, Аналитика → `/analytics`, Календарь → `/calendar`
- Данные: загружает профиль через `ProfileService.getProfile()`

### TodayScreen
Показывает программу на сегодня.

- Нет тренировки → «Сегодня нет запланированной тренировки» + ссылка на программы
- Есть тренировка:
  - Название, список упражнений (порядок, название, подходы × повторения)
  - Кнопка **Начать тренировку** → `TrainingService.getOrCreateTodaySession()` → `/session/{id}`

## Программы тренировок

### WorkoutsScreen
Два таба:

**«Мои программы»**
- Список созданных пользователем программ
- Кнопка **Создать программу** → `/workouts/create`
- Карточка программы: название + «X тренировок в неделю»
- Нажатие → `/workouts/{id}/exercises`

**«Стандартные»**
- 4 готовых шаблона (подробнее в разделе [Стандартные программы](#стандартные-программы))
- Нажатие → превью с упражнениями
- Кнопка **Использовать эту программу** → копирует программу со всеми упражнениями в «Мои программы»

### CreateWorkoutScreen
Форма создания программы:

- Поле «Название» (например, «Ноги», «Грудь+бицепс»)
- Выбор дней недели (Пн–Вс, множественный выбор)
- Слайдер **Длительность цикла** (4–16 недель, шаг 1)
  - Метка со значением отображается над ползунком
  - Двойное нажатие → ручной ввод (можно выйти за диапазон, например 20 недель)
- Кнопка **Далее** → `WorkoutService.createWorkout()` → `/workouts/{id}/exercises`
- Валидация: обязательное название и хотя бы один день

### AddExercisesScreen
Экран настройки упражнений в программе.

**Шапка**
- Название программы
- Иконка редактирования (карандаш) → диалог редактирования (название, дни, цикл)

**Секция «В программе»**
- Список добавленных упражнений с drag-and-drop для изменения порядка
  - Рукоятка перетаскивания (⠿) слева
  - `WorkoutService.reorderExercises()` при отпускании
- Нажатие на упражнение → нижний лист настроек:
  - Количество подходов (±1, минимум 1)
  - Количество повторений (текстовое поле)
  - Целевой вес (кг, необязательно)
  - Слайдер **Отдых** (0–120 сек, двойное нажатие → ручной ввод без ограничений)
  - Сохранение → `WorkoutService.updateWorkoutExercise()`
- Иконка корзины → удаление → `WorkoutService.removeExerciseFromWorkout()`

**Секция «Добавить упражнение»**
- Поле поиска (фильтрует по названию)
- Список всех стандартных упражнений
- Нажатие «+» → нижний лист настроек (аналогичен редактированию, дефолты: 3 подхода, 8–12 повт., 90 сек)
  - Сохранение → `WorkoutService.addExerciseToWorkout()`

## Сессия тренировки

### WorkoutSessionScreen
Основной рабочий экран во время тренировки.

**Навигация по упражнениям**
- Счётчик «X / Y» (текущее / всего)
- Название упражнения крупно
- «X из Y подходов выполнено»

**Управление весом**
- Отображение текущего веса (кг)
- Кнопки −2,5 / +2,5 кг

**Таблица подходов**
Для каждого подхода:
- Номер подхода (бейдж)
- Счётчик повторений (±1, редактируемое поле)
- Счётчик RPE (0–10, «—» для пустого)
- Кнопка-чекбокс → отмечает подход выполненным:
  - Немедленно сохраняет `TrainingService.saveSet()`
  - Запускает таймер отдыха

**Кнопки действий**
- **Добавить подход** — добавляет строку в таблицу
- **Следующее упражнение** — появляется когда все подходы выполнены и это не последнее упражнение

**Экран отдыха**
- Круговой прогресс-таймер (формат ММ:СС)
- Продолжительность берётся из настроек программы (`restSeconds`)
- Кнопка **Пропустить** — немедленный переход к следующему упражнению
- По истечении — автоматический переход

**Выход**
- Кнопка «×» → диалог подтверждения «Прервать тренировку?»
  - **Продолжить** — возврат в сессию
  - **Выйти** — останавливает `activeSessionProvider`, переход на `/home`

### SessionSummaryScreen
Экран итогов после остановки тренировки.

**Шапка**
- Иконка трофея, «Тренировка завершена!»
- Плитки: общее время · количество упражнений · количество подходов

**Таблица результатов**
- По каждому упражнению: карточка с упражнением и строками по подходам
- Строка подхода: [Номер] [Вес × Повт.] [RPE, если есть] [Иконка редактирования]
- Нажатие на строку → нижний лист редактирования:
  - Поля «Вес (кг)» и «Повторения»
  - Выбор RPE: чипы «—», 6, 7, 8, 9, 10
  - Кнопка **Применить**

**Сохранение**
- Кнопка **Сохранить тренировку**:
  1. Сохраняет все изменённые подходы через `TrainingService.updateSet()`
  2. Помечает сессию завершённой: `TrainingService.completeSession()`
  3. Сбрасывает `activeSessionProvider`
  4. Переход на `/home`
- Кнопка «×» / назад → диалог «Выйти без сохранения?»

## Календарь

### CalendarScreen
- Навигация по месяцам (← →)
- Виджет `TableCalendar` (локаль: русский, неделя с понедельника)

**Цветовое кодирование дат**
| Индикатор | Значение |
|---|---|
| Заливка синим | Выбранная дата |
| Серая заливка | Сегодня |
| Синяя точка под датой | Тренировка выполнена |
| Полупрозрачная точка | Тренировка запланирована |

**Легенда** — «Выполнено» / «Запланировано»

**Взаимодействие**
- Одиночное нажатие → выбирает дату, показывает список тренировок ниже
- **Двойное нажатие** → нижний лист:
  - Список тренировок на этот день (нажатие → `/workouts/{id}/exercises`)
  - Кнопка **Создать программу** (для будущих или пустых дат) → `/workouts/create`

**Данные**
- Прошедшие и завершённые сессии: `TrainingService.getSessionsByDateRange()`
- Будущие запланированные даты: вычисляются из цикла программы (дни недели + длительность цикла)

## Аналитика

### AnalyticsScreen
- Приветствие «Привет, [имя]!»
- Отображение цели пользователя («Твоя цель: …»)

**Карточка стрика**
- 🔥 «Стрик: X дней» (последовательные дни тренировок)
- «Тренировок всего: Y»

- `AnalyticsService.getWorkoutsThisWeek()` — сессий за текущую неделю
- `AnalyticsService.getVolumeThisWeek()` — объём за текущую неделю

## Профиль

### ProfileScreen
- Аватар (fallback — первая буква имени)
- Отображаемое имя (имя + фамилия, или никнейм, или email)
- Никнейм (@...)

**Личные данные**
- Имя, Фамилия, Отчество, Никнейм, Пол, Дата рождения, Город, Email, Телефон
- Кнопка **Редактировать**

**Статистика** (на главном профиле)
- Всего тренировок, лучший стрик (берётся из `AnalyticsService`)

**Настройки**
- Единицы: кг / фунты
- Тёмная тема
- Уведомления
*(переключатели присутствуют в UI, логика пока не реализована)*

**Кнопка «Выйти»** → `AuthService.signOut()` → `/`

### EditProfileScreen
Полноценная форма редактирования профиля.

**Аватар**
- Нажатие → нижний лист с сеткой стандартных аватаров или «Загрузить из галереи»
- Выбор стандартного → немедленное сохранение
- Загрузка фото → сжатие до 512px → `ProfileService.uploadAvatar()` → публичная ссылка в `profiles.avatar_url`

**Поля формы**
- Имя, Фамилия, Отчество
- Никнейм (3–32 символа, проверка доступности в реальном времени)
- Пол (чипы М / Ж)
- Дата рождения (ДД.ММ.ГГГГ + иконка-календарь)
- Город (TypeAhead с автодополнением через Dadata API)
- Email, Телефон

**Сохранение**
- `ProfileService.updateProfile()` — обновляет все поля
- Если изменён email → `ProfileService.updateEmail()` → Supabase отправляет письмо с подтверждением

### AuthService
Обёртка над Supabase Auth.

| Метод | Описание |

### ProfileService
Управление профилем пользователя.

| Метод | Описание |
|---|---|
| `getProfile()` | Профиль текущего пользователя |
| `createProfile(profile)` | Upsert нового профиля |
| `updateProfile(updates)` | Обновление полей профиля |
| `isNicknameAvailable(nickname)` | Проверка уникальности никнейма |
| `updateEmail(newEmail)` | Смена email в Auth и profiles |

### WorkoutService
CRUD для программ тренировок.

| Метод | Описание |
|---|---|
| `getMyWorkouts()` | Список программ пользователя |
| `createWorkout(name, days, cycleWeeks)` | Создание программы |
| `getWorkout(id)` | Программа по ID |
| `updateWorkout(id, ...)` | Изменение названия/дней/цикла |
| `addExerciseToWorkout(workoutId, exerciseId, ...)` | Добавление упражнения |
| `getWorkoutExercises(workoutId)` | Упражнения программы (с деталями) |
| `reorderExercises(workoutId, ids)` | Изменение порядка упражнений |
| `removeExerciseFromWorkout(id)` | Удаление упражнения из программы |
| `updateWorkoutExercise(id, ...)` | Изменение настроек упражнения |

### TrainingService
Управление сессиями и подходами.

| Метод | Описание |
|---|---|
| `getTodayWorkout()` | Программа на сегодняшний день недели |
| `getWorkoutExercisesForToday(workoutId)` | Упражнения для сегодняшней тренировки |
| `createSession(workoutId)` | Создание сессии на сегодня |
| `getOrCreateTodaySession(workoutId)` | Получить или создать сессию |
| `completeSession(sessionId)` | Пометить сессию выполненной |
| `getSessionsByDateRange(from, to)` | Сессии за период |
| `saveSet(sessionId, weId, setNumber, ...)` | Запись подхода |
| `getSessionSets(sessionId)` | Все подходы сессии |
| `updateSet(setId, weight, reps, rpe)` | Редактирование подхода |

### ExerciseService

| Метод | Описание |
|---|---|
| `getExercises({search})` | Все стандартные упражнения с опциональным поиском |

### AnalyticsService

| Метод | Описание |
|---|---|
| `getTotalWorkouts()` | Всего завершённых сессий |
| `getBestStreak()` | Наибольшая серия последовательных дней |
| `getWorkoutsThisWeek()` | Тренировок за текущую неделю |
| `getVolumeThisWeek()` | Объём (кг) за текущую неделю |
| `getExerciseMaxWeight(exerciseId)` | Макс. вес по датам для упражнения |

### PinService
PIN-аутентификация с защитой от брутфорса.

| Метод | Описание |
|---|---|
| `hasPin()` | Проверяет локальное хранилище, потом Supabase metadata |
| `setupPin(pin, userId)` | HMAC-SHA256 хэш → SecureStorage + Supabase metadata |
| `verifyPin(pin)` | Сверяет хэш |
| `clearPin()` | Удаляет PIN везде |
| `getFailedAttempts()` | Счётчик неудачных попыток |
| `incrementFailed()` | +1 к счётчику |
| `resetFailed()` | Сброс счётчика |
| Константа `maxAttempts` | 5 попыток до блокировки |

Хэширование: HMAC-SHA256(PIN, userId) — PIN подписывается идентификатором пользователя, что делает хэш уникальным на каждый аккаунт.

PIN синхронизируется между устройствами через `user_metadata` в Supabase Auth — при входе на новом устройстве хэш скачивается и сохраняется локально.

### CityService
Автодополнение городов.

| Метод | Описание |
|---|---|
| `suggest(query)` | Запрос к Dadata API, возвращает список городов |

## Модели данных

### Exercise
```dart
id, name, category (chest|back|legs|shoulders|arms|cardio),
```

## Управление состоянием

### activeSessionProvider
`StateNotifierProvider<ActiveSessionNotifier, ActiveSessionState>`

**Состояние `ActiveSessionState`:**
```dart
sessionId: String?
workoutId: String?
workoutName: String?
startTime: DateTime?

// Вычисляемые:
isActive     → sessionId != null
elapsed      → DateTime.now().difference(startTime) или Duration.zero
elapsedFormatted → "MM:SS" или "H:MM:SS"
```

**Методы нотификатора:**
- `start(sessionId, workoutId, workoutName)` — запуск сессии
- `stop()` — сброс состояния

Используется в `MainShell` (FAB), `WorkoutSessionScreen`, `SessionSummaryScreen`.

## Ключевые сценарии использования

### Регистрация и первый вход
```
Добро пожаловать
  → Создать аккаунт (никнейм + пароль)
  → [Настройка PIN, если "Запомнить меня"]
  → Онбординг: пол/возраст/вес → цель → уровень
  → Главный экран
```

### Повторный вход
```
Добро пожаловать
  → Если PIN настроен → PIN-экран → Главный экран
  → Иначе → Вход по паролю → Главный экран
```

### Создание программы тренировок
```
Программы → Создать программу
  → Название + дни недели + длительность цикла
  → Экран упражнений: добавить из библиотеки, настроить подходы/повт./отдых
  → Изменить порядок перетаскиванием
```

### Проведение тренировки
```
Главная → Тренировка на сегодня → Начать
  или FAB ▶ (если программа назначена на сегодня)
  → Для каждого упражнения:
      Установить вес → Отметить подходы (повт. + RPE) → Отдых → Следующее
  → Стоп → Итоги: проверить/отредактировать данные → Сохранить
```

### Просмотр истории
```
Календарь → Нажать дату → Список тренировок
Аналитика → Стрик, всего тренировок, объём за неделю
```
